# Idempotent Tool Execution

ツール実行の冪等性を確保し、安全な再試行を保証するためのパターン集。

## 1. 冪等性分類

ツール操作を以下の3カテゴリに分類し、再試行方針を決定する:

| カテゴリ | 例 | 再試行 | 対策 |
|---------|-----|--------|------|
| **自然冪等** | ファイル書き込み（同一内容）、`git status` | 安全 | そのまま再実行 |
| **条件付き冪等** | ファイル作成（既存時スキップ）、`mkdir -p` | 条件付き | 事前チェック追加 |
| **非冪等** | ファイル追記、カウンタ増分、`git commit` | 危険 | ガード処理が必須 |

## 2. 冪等性確保パターン

### ファイル操作

```
書き込み前
  ├─ 新規作成 → Write（上書きモード）→ 冪等
  ├─ 既存編集 → Edit（old_string 一致時のみ適用）→ 条件付き冪等
  └─ 追記     → 追記前に内容を確認 → 重複行があればスキップ
```

### コマンド実行

| 操作 | 非冪等パターン | 冪等パターン |
|------|--------------|-------------|
| ディレクトリ作成 | `mkdir dir` | `mkdir -p dir` |
| ファイル削除 | `rm file` | `rm -f file`（存在確認不要） |
| パッケージ追加 | 二重追加 | 追加前に依存リスト確認 |
| git コミット | 空コミット生成 | `git status` で変更有無を事前確認 |

## 3. 副作用チェックフロー

ツール実行前に以下を検証する:

```
実行予定のツール
  ├─ 読み取り専用か？ → Yes → 安全（そのまま実行）
  ├─ No → 副作用の範囲は？
  │   ├─ working_dir 内のみ → 許可
  │   ├─ プロジェクト外 → 停止・報告
  │   └─ 外部サービス → 停止・報告
  └─ 再試行時に副作用が重複するか？
      ├─ Yes → ガード処理を追加（§2参照）
      └─ No → そのまま実行
```

## 4. dry-run パターン

破壊的操作の前に影響を確認する:

| 操作 | dry-run 手法 |
|------|-------------|
| 大量ファイル削除 | 対象リストを先に出力・確認 |
| git clean | `git clean -n`（dry run）で確認後実行 |
| 一括置換 | Grep で対象箇所を先に確認 |
| テスト実行 | `--list` 等でテスト一覧を先に確認 |

## 5. retry_safe 判定との連携

結果報告時の `--no-retry-safe` 判断基準:

| 状態 | retry_safe | 理由 |
|------|-----------|------|
| 全操作が冪等 | Yes（デフォルト） | 再実行しても同じ結果 |
| 非冪等操作あり + ガード済み | Yes | ガードが重複防止 |
| 非冪等操作あり + ガードなし | No | 副作用が重複する恐れ |
| 部分的変更がリポジトリに残存 | No + `--partial-changes` | 中途半端な状態 |
